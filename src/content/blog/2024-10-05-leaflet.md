去年有參與開發一個 Leaflet 圖台，那時匆匆忙忙，再加上沒經驗，很多地方都沒有搞懂，程式能動就怎麼幹。今年圖台要新增兩個圖層，剛好我可以再回頭來研究一下 Leaflet 的 `L.geoJSON()`。

## 甚麼是 geoJSON

維基百科的解釋如下：

> geoJSON 是一種基於 JSON 的地理空間數據交換格式，它定義了幾種類型的 JSON 物件以及它們組合在一起的方法，以表示有關地理要素、屬性和它們的空間範圍的數據。

geoJSON 定義的地理資料類型（`type`）大致有以下幾種：

### 基本幾何圖形

- 點（Point）
- 線（LineString）
- 面（Polygon）

### 複合幾何圖形

- 多點（MultiPoint）
- 多線（MultiLineString）
- 多面（MultiPolygon）

geoJSON 的資料結構十分簡單，它是一個物件，通常會有 `type` 跟 `coordinates` 這兩個屬性：

- `type`：定義點、線、面的資料類型
- `coordinates`：座標陣列

要注意的是：**經度在前，緯度在後**。

### 點（Point）

```json
{
  "type": "Point",
  "coordinates": [30, 10]
}
```

### 線（LineString）

```json
{
  "type": "LineString",
  "coordinates": [
    [30, 10],
    [10, 30],
    [40, 40]
  ]
}
```

### 多邊形（Polygon）

```json
{
  "type": "Polygon",
  "coordinates": [
    [[30, 10], [40, 40], [20, 40], [10, 20], [30, 10]]
  ]
}
```

### 挖洞的多邊形

因為第二個陣列裡的座標位置在第一個陣列的座標範圍裡面，所以畫出來的面會像被挖了一個洞。

```json
{
  "type": "Polygon",
  "coordinates": [
    [[35, 10], [45, 45], [15, 40], [10, 20], [35, 10]],
    [[20, 30], [35, 35], [30, 20], [20, 30]]
  ]
}
```

以上是 geoJSON 的簡單說明，實際的結構可以更複雜。下面將針對如何在 Leaflet 中使用 `L.geoJSON`，把資料展現在地圖上做說明。

## `L.geoJSON` 如何使用

在 Leaflet 中，可以使用它提供的 `L.geoJSON` 方法，將點、線、面畫到地圖上。

`L.geoJSON()` 的第一個參數是 **GeoJSON 物件**，定義空間資訊；第二個參數也是一個物件，可以放入它提供的 `style`、`pointToLayer`、`onEachFeature` 及 `filter` 選項。

```js
const highBuilding = L.geoJSON(GeoJSON空間資料, {
  style: 線段及面的樣式
}).addTo(map);
```

## 如何畫一條線

首先定義一個簡單的線段 geoJSON 物件，`type` 是 `LineString`，`coordinates` 為一個陣列，裡面有三個經緯度座標。

```js
const lineJSON = {
  type: "LineString",
  coordinates: [
    [121.56, 25.033],
    [121.58, 25.015],
    [121.61, 25.043]
  ]
};
```

接著定義線段樣式：

```js
const lineStyle = {
  color: "#ff0000",
  // weight 控制線的粗細
  weight: 8,
  // opacity 控制透明度
  opacity: 0.65
};
```

最後使用 `L.geoJSON()` 把線段展在地圖上：

```js
const highBuilding = L.geoJSON(lineJSON, {
  style: lineStyle
}).addTo(map);
```

### 畫多條線

如果你想要畫兩條以上的線，可以把 geoJSON 改成陣列包住多個線段物件：

```js
const lineJSON = [
  {
    "type": "LineString",
    "coordinates": [[121.54, 25.033], [121.56, 25.033], [121.57, 25.033]]
  },
  {
    "type": "LineString",
    "coordinates": [[121.54, 25.030], [121.56, 25.030], [121.57, 25.030]]
  }
];
```

## 如何畫一個面

先定義一個面的 geoJSON 物件，`type` 是 `Polygon`，`coordinates` 是一個三層的陣列。

```js
const polygonJSON = {
  type: "Polygon",
  coordinates: [
    [
      [121.56, 25.033],
      [121.58, 25.015],
      [121.61, 25.043],
      [121.56, 25.043]
    ],
    [
      [121.52, 25.033],
      [121.48, 25.015],
      [121.48, 25.043],
      [121.52, 25.043]
    ]
  ]
};
```

定義面的樣式：

```js
const polygonStyle = {
  color: "#ff0000",
  // weight 控制線的粗細
  weight: 8,
  // opacity 控制透明度
  opacity: 0.65,
  // 如果沒有 fillColor，則由 color 控制填色
  fillColor: "#B0DE5C",
  fillOpacity: 0.8
};
```

最後使用 `L.geoJSON()` 把面展在地圖上：

```js
const polygons = L.geoJSON(polygonJSON, {
  style: polygonStyle
}).addTo(map);
```

## 如何畫一個點

先定義點的 geoJSON：

```js
const pointJSON = {
  "type": "Point",
  "coordinates": [121.5647238, 25.0336659]
};
```

如果不自訂樣式，可以直接使用預設的 `marker`：

```js
const highBuilding = L.geoJSON(pointJSON).addTo(map);
```

如果要自訂點的樣式，就要在第二個參數中使用 `pointToLayer`，回傳 `L.circleMarker` 或 `L.marker`。

```js
const highBuilding = L.geoJSON(taipei101, {
  pointToLayer(feature, latlng) {
    return L.circleMarker(latlng, {
      // 控制點的大小
      radius: 8,
      // ORANGE
      fillColor: "#ff7800",
      // 線框顏色
      color: "#000",
      // 線框粗細
      weight: 1,
      opacity: 1,
      fillOpacity: 0.8
    });
  }
}).addTo(map);
```

## 更複雜多樣的 geoJSON

geoJSON 也可以是下面這種結構，外層的 `type` 設成 `Feature`，在 `geometry` 放入原本的 `type`（例如 `Point`、`LineString`、`Polygon`）與 `coordinates`，另外也可以在 `properties` 中放樣式或 popup 內容。

```js
const campus = {
  "type": "Feature",
  "properties": {
    "popupContent": "This is the Auraria West Campus",
    "style": {
      weight: 2,
      color: "#999",
      opacity: 1,
      fillColor: "#FF0000",
      fillOpacity: 0.8
    }
  },
  "geometry": {
    "type": "MultiPolygon",
    "coordinates": [
      [
        [
          [121.519708, 25.035685],
          [121.523249, 25.035685],
          [121.523249, 25.033586],
          [121.519708, 25.033586]
        ]
      ]
    ]
  }
};
```

也可以將外層的 `type` 設成 `FeatureCollection`，在內層的 `features` 陣列中，放入多個 `Feature` 物件。

```js
const bicycleRental = {
  "type": "FeatureCollection",
  "features": [
    {
      "geometry": {
        "type": "Point",
        "coordinates": [121.5647238, 25.0336659]
      },
      "type": "Feature",
      "properties": {
        "popupContent": "台北i101"
      },
      "id": 51
    }
  ]
};
```

當 `type` 設為 `FeatureCollection` 的時候，可以同時包含各種點、線及面的空間資訊，`L.geoJSON` 會自動判斷是點還是面。

```js
const bicycleRental = {
  "type": "FeatureCollection",
  "features": [
    {
      "geometry": {
        "type": "Point",
        "coordinates": [121.5647238, 25.0336659]
      },
      "type": "Feature",
      "properties": {
        "popupContent": "台北i101"
      },
      "id": 51
    },
    {
      "geometry": {
        "type": "Point",
        "coordinates": [121.5848238, 25.0337659]
      },
      "type": "Feature",
      "properties": {
        "popupContent": "This is a B-Cycle Station. Come pick up a bike and pay by the hour. What a deal!"
      },
      "id": 52
    },
    {
      "geometry": {
        "type": "Polygon",
        "coordinates": [[
          [121.55, 25.04],
          [121.6, 25.05],
          [121.6, 25.055],
          [121.55, 25.055]
        ]]
      },
      "type": "Feature",
      "properties": {
        "popupContent": "This is a B-Cycle Station. Come pick up a bike and pay by the hour. What a deal!"
      },
      "id": 54
    }
  ]
};
```

## `L.geoJSON` 提供的四種選項方法

搭配這四種選項方法，geoJSON 的資料屬性除了 `type`、`coordinates` 之外，也可以加入其他屬性來使用。

## `style`

`style` 用來定義線跟面的樣式，作為 `L.geoJSON()` 第二個參數的物件屬性。

以上面的 `Polygon` 為例：

```js
const polygonStyle = {
  color: "#ff0000",
  // weight 控制線的粗細
  weight: 8,
  // opacity 控制透明度
  opacity: 0.65,
  // 如果沒有 fillColor 選項，則由 color 控制填色
  fillColor: "#B0DE5C",
  fillOpacity: 0.8
};

const polygons = L.geoJSON(polygonJSON, {
  style: polygonStyle
}).addTo(map);
```

`style` 也可以當成函式來使用，以下是官網範例：

```js
var states = [{
  "type": "Feature",
  "properties": {"party": "Republican"},
  "geometry": {
    "type": "Polygon",
    "coordinates": [[
      [-104.05, 48.99],
      [-97.22, 48.98],
      [-96.58, 45.94],
      [-104.03, 45.94],
      [-104.05, 48.99]
    ]]
  }
}, {
  "type": "Feature",
  "properties": {"party": "Democrat"},
  "geometry": {
    "type": "Polygon",
    "coordinates": [[
      [-109.05, 41.00],
      [-102.06, 40.99],
      [-102.03, 36.99],
      [-109.04, 36.99],
      [-109.05, 41.00]
    ]]
  }
}];

L.geoJSON(states, {
  style: function(feature) {
    switch (feature.properties.party) {
      case "Republican":
        return { color: "#ff0000" };
      case "Democrat":
        return { color: "#0000ff" };
    }
  }
}).addTo(map);
```

## `onEachFeature`

`onEachFeature` 是一個函式，在每一個空間資訊要被加入 geoJSON 圖層的時候會被呼叫。常見用法是將 `properties` 中帶的資訊呈現在 popup 上。

```js
function onEachFeature(feature, layer) {
  // does this feature have a property named popupContent?
  if (feature.properties && feature.properties.popupContent) {
    layer.bindPopup(feature.properties.popupContent);
  }
}

var geojsonFeature = {
  "type": "Feature",
  "properties": {
    "name": "Coors Field",
    "amenity": "Baseball Stadium",
    "popupContent": "This is where the Rockies play!"
  },
  "geometry": {
    "type": "Point",
    "coordinates": [-104.99404, 39.75621]
  }
};

L.geoJSON(geojsonFeature, {
  onEachFeature: onEachFeature
}).addTo(map);
```

## `pointToLayer`

如果要把 geoJSON 中的點資訊畫在地圖上，但不使用預設的 `marker`，就必須使用 `pointToLayer`。

```js
var geojsonMarkerOptions = {
  radius: 8,
  fillColor: "#ff7800",
  color: "#000",
  weight: 1,
  opacity: 1,
  fillOpacity: 0.8
};

L.geoJSON(someGeojsonFeature, {
  pointToLayer: function(feature, latlng) {
    return L.circleMarker(latlng, geojsonMarkerOptions);
  }
}).addTo(map);
```

## `filter`

`filter` 可以用來過濾 geoJSON 的屬性，常常用來控制圖徵是否顯示。

下面這個例子中，`Busch Field` 因為 `show_on_map` 屬性為 `false`，所以不會顯示在地圖上。

```js
var someFeatures = [{
  "type": "Feature",
  "properties": {
    "name": "Coors Field",
    "show_on_map": true
  },
  "geometry": {
    "type": "Point",
    "coordinates": [-104.99404, 39.75621]
  }
}, {
  "type": "Feature",
  "properties": {
    "name": "Busch Field",
    "show_on_map": false
  },
  "geometry": {
    "type": "Point",
    "coordinates": [-104.98404, 39.74621]
  }
}];

L.geoJSON(someFeatures, {
  filter: function(feature, layer) {
    return feature.properties.show_on_map;
  }
}).addTo(map);
```
