它的行為取決於是否有提供**比較函式**（compare function）。

## Array.prototype.sort() 方法的運作原理

`sort()` 方法用於對 JavaScript 陣列**就地排序**，也就是會**直接修改原陣列**，並回傳排序後的陣列。  
它的行為取決於是否有提供**比較函式**（compare function）。

---

## 1. 沒有比較函式時：`sort()`

如果沒有提供比較函式，`sort()` 預設會先把陣列元素**轉成字串**，再依照 **Unicode 字典順序**（lexicographic order）排序。

### 範例

```js
const arr = [100, 2, 50, 20, 5];
arr.sort();
console.log(arr);
// ➝ [100, 2, 20, 5, 50]
```

### 為什麼結果不如預期？

因為 `sort()` 預設是按照**字串的 Unicode 值**排序，而不是依照數字大小排序。

例如這些值會先變成字串：

- `100`
- `2`
- `50`
- `20`
- `5`

因此比較時其實是在比：

- `"100"`
- `"2"`
- `"50"`
- `"20"`
- `"5"`

所以才會得到：

```js
[100, 2, 20, 5, 50]
```

也就是：

```text
"100" < "2" < "20" < "5" < "50"
```

---

## 2. 提供比較函式：`sort(compareFn)`

如果要正確排序數值，就必須提供比較函式（compare function）。

比較函式應該回傳：

- **負數（< 0）**：`a` 排在 `b` 前面
- **零（0）**：保持原本順序
- **正數（> 0）**：`a` 排在 `b` 後面

---

## 數字排序

### 範例

```js
const arr = [100, 2, 50, 20, 5];
arr.sort((a, b) => a - b);
console.log(arr);
// ➝ [2, 5, 20, 50, 100]
```

### 如何運作？

```js
(a, b) => a - b
```

這代表：

- `a - b < 0`：`a` 會排在 `b` 前面
- `a - b > 0`：`a` 會排在 `b` 後面

例如：

- `100 - 2 > 0` → `2` 排在 `100` 前面
- `2 - 50 < 0` → `2` 排在 `50` 前面
- `50 - 20 > 0` → `20` 排在 `50` 前面

依此類推，直到排序完成。

---

## `sort((a, b) => a - b)` 遇到字串時會怎樣？

### 1. 數字型字串

如果字串內容是數字，例如：

```js
["10", "5", "30"]
```

那麼在做 `a - b` 時計算時，JavaScript 會自動把它們轉成數字，因此通常仍可正常排序。

### 範例

```js
const arr = ["10", "5", "30"];
arr.sort((a, b) => a - b);
console.log(arr);
// ➝ ["5", "10", "30"]
```

---

### 2. 非數字字串

如果是一般字串，例如：

```js
["apple", "banana", "cherry"]
```

做 `a - b` 時會轉換失敗，結果變成 `NaN`。

### 範例

```js
const fruits = ["apple", "banana", "cherry"];
fruits.sort((a, b) => a - b);
console.log(fruits);
// 可能無法得到穩定且可預期的結果
```

### 原因

因為：

- `"apple" - "banana"` → `NaN`
- `NaN` 參與比較時，無法保證正確排序
- 不同 JavaScript 引擎可能會有不同表現

---

## 字串排序

對字串進行排序時，建議使用 `localeCompare()`。

### 範例

```js
const names = ["Banana", "apple", "Cherry"];
names.sort((a, b) => a.localeCompare(b));
console.log(names);
// ➝ ["apple", "Banana", "Cherry"]
```

### 好處

`localeCompare()` 會依照語系規則比較字串，排序結果通常比直接用 `sort()` 更符合直覺。

例如：

- `a` 可以正確排在 `B` 前面
- 比較適合處理英文、中文或其他語系文字

---

## 重點整理

### `sort()` 預設行為

- 會直接修改原陣列
- 預設會先把元素轉成字串
- 再依 Unicode 字典順序排序

### 數字排序

建議寫法：

```js
arr.sort((a, b) => a - b);
```

### 字串排序

建議寫法：

```js
arr.sort((a, b) => a.localeCompare(b));
```

---

## 補充提醒

如果你不想修改原本的陣列，可以先複製一份再排序：

```js
const arr = [100, 2, 50, 20, 5];
const sortedArr = [...arr].sort((a, b) => a - b);

console.log(arr);       // 原陣列不變
console.log(sortedArr); // 排序後的新陣列
```
