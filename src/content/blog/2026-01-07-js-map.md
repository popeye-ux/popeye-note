JS 提供許多方法來處理陣列，`map()` 是常用的方法之一。下面整理幾種常見的使用情境與寫法，方便自己記憶與複習。

## 基本語法

`map()` 方法會對陣列中的每個元素進行處理，並回傳一個**長度相同的新陣列**，且**不會改變原始陣列**。

```js
let newArray = arr.map(function callback(currentValue[, index[, array]]) {
  // return element for newArray
}[, thisArg]);
```

---

## 使用情境一：資料轉換

將物件陣列中的某個屬性提取出來，組成新的陣列。

```js
const users = [
  { id: 1, name: 'Mary', email: 'mary@gmail.com' },
  { id: 2, name: 'John', email: 'john@gmail.com' }
];

const userName = users.map(user => user.name);
```

---

## 使用情境二：數值運算與修正

對陣列中的數值進行統一的數學運算，並回傳一個新陣列。  
例如：打折、單位換算。

```js
const prices = [100, 200, 300];

// 打 8 折
const discountPrice = prices.map(price => price * 0.8);
```

---

## 使用情境三：替物件新增或修改屬性

在不破壞原始資料的情況下，為每個物件新增屬性。

```js
const students = [
  { name: 'John', scores: 55 },
  { name: 'Mary', scores: 70 }
];

const gradedStudents = students.map(student => ({
  ...student,
  passed: student.scores >= 60
}));
```

---

## 使用情境四：搭配其他陣列方法

`map()` 常常會和 `filter()` 串聯使用，先用 `filter()` 篩選資料，再用 `map()` 轉換資料。

```js
const students = [
  { name: 'John', scores: 55 },
  { name: 'Mary', scores: 70 }
];

const result = students
  .filter(student => student.scores >= 60)
  .map(student => student.name);
```

---

## 使用小技巧與注意事項

| 特性 | 說明 |
|---|---|
| 回傳新陣列 | `map()` 不會改變原陣列（Immutability），這對除錯很有幫助。 |
| 長度一致 | 回傳的新陣列長度一定會等於原陣列長度。 |
| 避免副作用 | 盡量不要在 `map()` 裡執行 `console.log()` 或修改外部變數，那比較像是 `forEach()` 的用途。 |
| 一定要有 return | 如果忘記寫 `return`，新陣列的元素會變成 `undefined`。 |

---

## 重點整理

- `map()` 適合用來做**資料轉換**
- 它會回傳**新的陣列**
- 不會修改原本的陣列
- 很適合搭配 `filter()`、`reduce()` 一起使用
